Classification d'images



Récement de nombreuse applications à base de réseaux à convolution (CNN) sont apparut dans les entreprises, les objects connéctés, les applications mobiles... L'objectif de ce projet, est de voir comment construire un réseau de neurones pour classifier des images. Nous allons tester plusieurs méthodes pour former et optimiser l'apprentissage d'un CNN. Pour ce faire, nous allons utiliser un jeux de données composé d'images de chiens et de chats disponible sur la platform Kaggle. Le jeu de données est composé de 25 000 images labélisées et de 12500 images non labélisées permettant de d'optenir un score dans la competition kaggle.

Création du modèle

Il existe deux manières de créer un réseaux à convolution. La première est de le construire soit même, ou alors d'utiliser du transfert learning pour récupérer l'architechture et les poids d'un modèle pré-entrainé.
Nous allons tester les deux méthodes dans ce projet.

Construction d'un CNN

Il est tout à fait possible de creér sont propre réseaux à convolution. A la difference des réseaux de neurone simple, qui croisent toutes les features les une aux autres, les CNN eux, traitent des filtres d'image puis les relient au fure et à mesures que l'on avance dans le réseau. Cela permet d'avoir moins de parametre à optimiser. L'architecture d'un CNN est composé de convolution et de pooling. Les convolutions permettent de réaliser automatiquements des filtres sur une image. Les pooling permettent de résumé l'information de plusieurs neurone en un seul neurone. Cela permet de diminer les dimentions d'une image.

Notre CNN prend donc en entré une image de taille (112,112,3) et redonne en sortie un vecteur de une valeur en sortie. Cette valeur correspond à la probabilité qu'il y est un chat sur la photo. Nous pouvons maintenant entrainer le modèle.

Entrainement du modèle

Pour entrainer le modèle, nous devons redimentionner les images pour que celle ci soit toute à la même dimenssion. Nous avons choisi ici de donner en entrée des vecteurs de 112* 122* 3. Pour convertir les images en vecteur et les redimentionné, nous avons deux possibilités. Soit nous crééons un vecteur de taille (nb_images, 112, 112, 3) qui contient l'enssemble des images. L'inconvenient de cette méthode est que si le nombre d'image est élévé, vous risquer de saturer la RAM de votre ordinateur. Mais avec cette méthode, l'entrainement du modèle est bien plus rapide. Soit nous importons uniquement le batch d'image que nous envoyons en entré de notre CNN. Cette méthode permet de ne pas saturer la RAM mais augmente la durée de l'apprentissage du modèle. Ici, nous allons utiliser la deuxième méthode, car généralement dans ce genre de projet, le dataset d'image ne permet pas de chargé toute les images dans la RAM.

Ajout de regularisation

Drop Out

Dans un premier temps, nous allons rajouter du drop out dans notre modèle. Le drop out permet "supprimer" aléatoirement des neurones dans un layer pour chaque batch d'entrainement.

Transfert Learning

Comme nous l'avons évoqué plus haut, nous ne sommes pas obligé de conbstruire un CNN de zero. Il est possible récuperer l'architecture et les poids d'un modèle pé-entrainé sur un autre jeux de données. Généralement nous utilisons un jeux données entréné sur le jeu de données ImageNet, car avec ces 1000 classes, il permet de diposer d'un modèle assez généraliste.

pre_training_model = tf.keras.applications.InceptionResNetV2(weights='imagenet', include_top=False, input_shape=(112,112,3))

model = tf.keras.models.Sequential()
model.add(pre_training_model)
model.add(tf.keras.layers.Flatten())
model.add(tf.keras.layers.Dense(256, activation='relu'))
model.add(tf.keras.layers.Dense(1, activation='sigmoid'))

model.compile(loss='binary_crossentropy', optimizer=tf.keras.optimizers.Adam(), metrics=['acc'])

model.summary()
